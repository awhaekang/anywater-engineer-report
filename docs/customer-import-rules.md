# 애니워터 고객 데이터 분류 규칙

엔콤 고객 데이터 엑셀을 앱/Supabase로 가져올 때 사용하는 기본 분류 규칙입니다.

## 상태 분류

### 관리중

현장 엔지니어의 근처 설치처 추천에 기본 표시합니다.

```text
관리여부 = 관리함
제품상태 = 설치완료
계약해지일 비어있음
삭제일 비어있음
```

### 해지/회수

현장 추천에서는 기본 제외하고, 관리자 검색에서만 확인합니다.

```text
삭제일 있음
또는 계약해지일 있음
또는 제품상태 = 계약해지
또는 제품회수일 있음 + 관리여부 = 관리안함
```

### 확인 필요

자동 제외하지 않고 관리자 확인 대상으로 둡니다.

```text
해지일 있음
또는 CMS상태 = 해지완료
단, 제품상태/관리여부가 명확히 해지로 판단되지 않는 경우
```

`해지일`과 `CMS상태=해지완료`는 결제 변경 또는 CMS 해지일 수 있으므로 단독 기준으로 현장 추천에서 제외하지 않습니다.

### 임시 등록

현장에서 엔지니어가 직접 추가한 설치처입니다.

```text
source = field
customerStatus = temporary
```

관리자나 사무 담당자가 나중에 엔콤 고객 데이터와 대조해 정식 고객으로 병합하거나 삭제합니다.

## 앱 추천 기준

현장 엔지니어 기본 추천:

```text
customerStatus in (관리중, 임시 등록)
```

관리자 검색:

```text
관리중, 확인 필요, 해지/회수, 임시 등록 모두 조회 가능
```
