# 애니워터 현장 방문 보고 웹앱

정수필터 렌탈 설치처의 현장 방문 보고를 표준화하고, 고객사별 이력을 누적하기 위한 웹앱 프로토타입입니다.

## 새 채팅 인수인계

새 채팅에서 이어서 작업할 때는 먼저 아래 문서를 읽으면 됩니다.

[docs/HANDOFF.md](docs/HANDOFF.md)

## 현재 기능

- 위치 기반 근처 설치처 추천
- 실제 엔콤 고객데이터 로컬 로드
- 설치처별 누적 방문 이력
- 사무실 오더 생성
- 오늘 방문 오더
- 처리 제품/필터 다중 선택
- 다음 방문 재확인 항목
- 방문 유형별 체크리스트
- 방문 유형별 필수 사진 단계
- 관리자 보고서 검토/승인
- 관리자 설치처 등록
- 관리자 방문 유형 양식 추가
- 보고서 CSV 내보내기
- 네이버 밴드용 요약문 복사

## 로컬 확인

```bash
python3 -m http.server 4174 --bind 127.0.0.1
```

브라우저에서 `http://127.0.0.1:4174/index.html`을 엽니다.

현재 버전은 브라우저 저장소를 사용합니다. Supabase 연결 전까지는 같은 브라우저 안에서만 데이터가 유지됩니다.

실제 고객 데이터가 `data/customers.local.json`에 있으므로 로컬 서버는 반드시 `127.0.0.1`에만 바인딩하세요.

## Supabase 전환 방향

1. Supabase 프로젝트 생성
2. `supabase/schema.sql` 실행
3. 사진 저장용 Storage bucket 생성: `visit-photos`
4. 로그인 방식 결정: 이메일 로그인 또는 사내 계정 연동
5. `app.js`의 `localStorage` 저장 로직을 Supabase API 호출로 교체

핵심 원칙은 보고서를 긴 본문 하나로 저장하지 않는 것입니다. 설치처, 방문 유형, 체크 결과, 사진 단계, 승인 상태를 분리해야 나중에 통계와 회의 자료를 만들 수 있습니다.

## Vercel 배포 방향

정적 파일 배포부터 시작할 수 있습니다.

```bash
vercel
```

또는 Git 저장소를 Vercel에 연결하고 루트 디렉터리를 그대로 배포하면 됩니다.

Supabase 연동 후에는 Vercel 환경 변수에 다음 값을 둡니다.

```text
SUPABASE_URL=
SUPABASE_SERVICE_ROLE_KEY=
```

`SUPABASE_SERVICE_ROLE_KEY`는 서버 함수(`api/customers.js`)에서만 사용하고 브라우저 코드에는 노출하지 않습니다.
